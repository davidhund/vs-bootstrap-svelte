# Contributing

This is a personal starter template, but improvements are welcome. If you've found a bug, want to suggest a better pattern, or improve the docs:

## Before you start

- Check existing [issues](https://github.com/davidhund/vs-bootstrap-svelte/issues) and [discussions](https://github.com/davidhund/vs-bootstrap-svelte/discussions)
- Read [CLAUDE.md](./CLAUDE.md) to understand the conventions and philosophy
- Test your changes locally before opening a PR

## How to contribute

1. **Fork** this repo
2. **Create a branch** with a sensible name: `feat/add-x`, `fix/y`, `docs/improve-z`
3. **Make your change** following the conventions in [CLAUDE.md](./CLAUDE.md)
4. **Test locally:**
   ```sh
   pnpm install
   pnpm ci        # lint + check + unit tests
   pnpm dev       # verify the template works
   ```
5. **Commit** using [Conventional Commits](https://www.conventionalcommits.org/)
6. **Open a PR** with a clear title and description

## What we're looking for

✅ Bug fixes (especially in bootstrap flow, tooling setup, or docs)  
✅ Pattern improvements (cleaner examples, better conventions)  
✅ Documentation clarifications  
✅ Dependency updates (keep stack current)  

❌ Major feature additions (keep this lean and minimal)  
❌ Framework/tool swaps without discussion  

## Questions?

Use [GitHub Discussions](https://github.com/davidhund/vs-bootstrap-svelte/discussions) to ask questions or brainstorm ideas before opening an issue.
