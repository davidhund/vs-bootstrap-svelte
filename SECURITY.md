# Security Policy

## Reporting a vulnerability

**Do not** open a public GitHub issue for security vulnerabilities.

Instead, please report to the maintainer at `<owner-email>` with:
- Description of the vulnerability
- Affected versions
- Steps to reproduce (if possible)
- Potential impact

I will acknowledge receipt within 48 hours and work toward a fix and disclosure timeline.

## Scope

This policy applies to vulnerabilities in:
- The template code itself
- Tooling configuration
- Documentation that could lead to insecure setups

**Not in scope:**
- Vulnerabilities in upstream dependencies (report directly to maintainers)
- Issues with projects created *from* this template

## Known issues

See [GitHub Security Advisories](https://github.com/davidhund/vs-bootstrap-svelte/security/advisories) for any published advisories.
